CSS Web Developer - Posicionamentos e Exibição

Display: Flex

Display: Flex em CSS define que o elemento se tornará um container flexível, ou seja, um Flex Container. Com isso, todos os elementos filhos se tornam Flex Items e passam a ser posicionados de forma mais dinâmica e controlada.

Principais características:

  • Os itens ficam alinhados em uma única linha (por padrão).
  • É possível mudar o eixo principal (linha ou coluna).
  • Permite alinhamento e espaçamento fácil entre os itens.
  • Ideal para layouts responsivos e centralizações.

Propriedades:

flex-direction

É uma propriedade usada em um container flexível (ou seja, um elemento com display: flex).
para definir a direção principal em que os itens flexíveis serão organizados dentro do container.

display: flex;
flex-direction: row;
  • row (padrão): Os itens são dispostos em linha, da esquerda para a direita (padrão).
  • row-reverse: Os itens são dispostos em linha, mas da direita para a esquerda.
  • column: Os itens são dispostos em coluna, de cima para baixo.
  • column-reverse: Os itens são dispostos em coluna, mas de baixo para cima.
justify-content

A propriedade justify-content em CSS é usada para alinhar os itens ao longo do eixo principal dentro de um container flexível (display: flex).
Ou seja, ela controla como o espaço extra é distribuído entre os itens (ou ao redor deles), de acordo com a direção definida por flex-direction.

display: flex;
justify-content: center;
  • flex-start (padrão): Alinha os itens no início do eixo principal (padrão).
  • flex-end: Alinha os itens no final do eixo principal.
  • center: Centraliza os itens.
  • space-between: Distribui os itens com espaço igual entre eles (sem espaço nas bordas).
  • space-around: Distribui os itens com espaço igual ao redor de cada item (metade nas bordas).
  • space-evenly: Distribui os itens com espaço totalmente igual entre todos (incluindo bordas).
align-items

A propriedade align-items em CSS é usada para alinhar os itens ao longo do eixo transversal (ou eixo cruzado) dentro de um container flex (display: flex).
Enquanto justify-content alinha horizontalmente (no eixo principal), align-items alinha verticalmente (no eixo perpendicular) — a menos que o eixo principal tenha sido alterado com flex-direction.

display: flex;
align-items: center;
  • stretch (padrão): Os itens se esticam para preencher todo o eixo transversal.
  • flex-start: Alinha os itens ao início do eixo transversal.
  • flex-end: Alinha os itens ao final do eixo transversal.
  • center: Centraliza os itens no eixo transversal.
  • baseline: Alinha os itens pela linha de base do texto.
align-content

A propriedade align-content em CSS é usada para alinhar as linhas do conteúdo flexível ao longo do eixo transversal, somente quando há múltiplas linhas (ou seja, quando o flex container tem flex-wrap: wrap ou wrap-reverse).
Ela é parecida com align-items, mas enquanto align-items alinha os itens individuais, align-content alinha as linhas dos itens.

display: flex;
flex-wrap: wrap;
align-content: center;
flex-wrap

A propriedade flex-wrap em CSS define se os itens flexíveis devem ou não quebrar linha quando não houver espaço suficiente no container.
Por padrão, os itens em um container com display: flex ficam em uma única linha. Com flex-wrap, você pode permitir que eles “quebrem” para a próxima linha, como um wrap mesmo — tipo quando o texto pula para baixo ao final da linha.

display: flex;
flex-wrap: wrap;
  • nowrap (padrão): Todos os itens permanecem em uma única linha, mesmo que transbordem o container.
  • wrap: Os itens quebram para a próxima linha quando necessário (de cima para baixo).
  • wrap-reverse: Os itens também quebram, mas as linhas são empilhadas de baixo para cima (ordem reversa).
gap

A propriedade gap em CSS é usada para definir o espaçamento entre os itens dentro de containers com display: flex, grid ou inline-flex/grid.
Ela substitui a necessidade de usar margin-right, margin-bottom, etc., entre os itens — deixando o código mais limpo e fácil de manter.

display: flex;
gap: 16px;

Display: None

Em CSS, display: none; é uma propriedade usada para ocultar completamente um elemento da página, como se ele não existisse no HTML.
Ele não ocupa espaço no layout e não é renderizado no navegador, diferentemente de outras formas de esconder um elemento (como visibility: hidden, que esconde o conteúdo mas ainda ocupa espaço).

Características de um elemento com display: none;
  • Remove o elemento da renderização:

    • Ele não aparece visualmente nem ocupa espaço.

  • Não pode ser interagido:

    • O elemento deixa de fazer parte do fluxo do documento.

  • Pode ser reexibido dinamicamente:

    • Usando JavaScript ou alterando a regra de CSS, é possível "trazê-lo de volta".

display: none;

Com isso, um elemento com a classe .menu ficará totalmente invisível e fora do layout da página.

Display: Block

No CSS, a propriedade display: block; é usada para definir um elemento como um bloco, o que significa que o elemento ocupará toda a largura disponível em seu contêiner pai (por padrão) e terá uma quebra de linha antes e depois dele.
Isso faz com que o elemento seja exibido em uma nova linha, empurrando qualquer outro conteúdo que esteja ao lado para baixo.

Características de um elemento com display: block;
  • Ocupação de largura total:

    • Por padrão, o elemento ocupa 100% da largura do seu contêiner pai.

  • Quebra de linha:

    • Um elemento com display: block; começa em uma nova linha e empurra outros elementos para baixo.

  • Pode ter largura e altura definidas:

    • Diferente de elementos em linha, como "span", elementos de bloco podem ter propriedades de width e height ajustadas.

display: block;

Neste exemplo, o elemento "div" será exibido como um bloco. Elementos como "div", "p", "h1" até "h6", e "section" já têm o comportamento de bloco por padrão.

Esse comportamento é útil para organizar a estrutura da página de maneira mais clara e para quando você precisa de um elemento que "quebre" a linha para criar um layout mais organizado.

Display: Inline

Display: inline é um valor da propriedade display no CSS que faz com que o elemento se comporte como parte de uma linha de texto, como se fosse uma palavra ou letra dentro de um parágrafo.

Comportamento do Inline
  • Quebra de linha: ❌ Não quebra linha automaticamente.
  • Largura e altura definidas: ❌ Não aceita width ou height (essas propriedades são ignoradas).
  • Padding e margin: ✅ Só horizontal (esquerda e direita) funciona totalmente. Topo e base são limitados.
  • Fluxo do documento: ✅ Fica na mesma linha que outros elementos inline, se houver espaço.
  • Exemplos nativos: "span", "a", "strong", "em"
Isso é um elemento inline no meio do texto.

Esse span não vai quebrar a linha, e vai se comportar como parte do texto corrido.

Display: Inline-Block

Display: inline-block é um tipo de exibição que combina características dos elementos inline e block.

Comportamento do Inline-Block
  • Quebra de linha automática: ❌ Não (fica ao lado de outros inline ou inline-block, se couber)
  • Aceita width e height: ✅ Sim
  • Aceita margin e padding: ✅ Sim, completamente
  • Fluxo do documento: ✅ Fica em linha, mas com comportamento de bloco

Display: Grid

O Display: Grid em CSS é uma propriedade que transforma um elemento em um container de grid, permitindo organizar seus elementos filhos em linhas e colunas de forma precisa e controlada. É uma das formas mais poderosas de criar layouts responsivos e complexos com CSS.

Vantagens do Grid:

  • Controle total sobre linhas e colunas
  • Suporte a alinhamento vertical e horizontal
  • Criação de layouts responsivos com facilidade
  • Pode ser combinado com outras técnicas como Flexbox

Propriedades:

grid-template-columns

A propriedade grid-template-columns em CSS é usada para definir o número de colunas e o tamanho de cada uma dentro de um container com display: grid.

grid-template-columns: valor1 valor2 valor3 ...;
  • px: pixels
  • %: porcentagem
  • fr: significa "fração do espaço disponível".

    • É uma unidade exclusiva do CSS Grid usada para dividir o espaço restante entre colunas (ou linhas) de forma proporcional.

    grid-template-columns: 1fr 2fr 1fr;

    O espaço total disponível é dividido em 4 partes:

    • A primeira coluna recebe 1 parte.
    • A segunda recebe 2 partes.
    • A terceira recebe 1 parte.
  • auto: tamanho automático baseado no conteúdo
  • min-content:

    • Define o tamanho mínimo necessário para que o conteúdo não quebre ou transborde.
    O elemento só vai ocupar o espaço que ele realmente precisa (por exemplo, o menor tamanho de uma palavra sem quebrar).

    grid-template-columns: min-content;

    Se o conteúdo for uma palavra longa, ele só vai ter largura suficiente pra caber essa palavra sem quebrar.

  • max-content:

    • Define o tamanho máximo que o conteúdo pode ocupar sem quebrar.
    Ele estica até mostrar todo o conteúdo em uma linha, sem cortar nem dividir.

    grid-template-columns: max-content;

    Se tiver várias palavras, ele vai esticar a célula até mostrar tudo em uma linha só, sem quebra.

  • min-content():

    • Define um intervalo de tamanho entre um mínimo e um máximo.

    grid-template-columns: minmax(200px, 1fr);

    “Essa coluna deve ter no mínimo 200px, mas pode crescer até ocupar 1fr do espaço, se sobrar.”

grid-template-rows

O grid-template-rows, é irmão do grid-template-columns. Como o nome já diz, ele controla as linhas do grid, em vez das colunas.
Pode ser utilizado com as mesmas unidades do grid-template-rows.

grid-template-rows: valor1 valor2 valor3 ...;
grid-template-areas

Grid Template Areas é uma propriedade do CSS Grid que permite criar layouts visuais mais intuitivos e legíveis, nomeando áreas da grade (grid) para facilitar o posicionamento dos elementos.

.container{
   grid-template-areas:
   "header header"
   "menu content"
   "footer footer";
}
      .header{ grid-area: header; }
      .menu{ grid-area: menu; }
      .content{ grid-area: content; }
      .footer{ grid-area: footer; }

A grid-template-areas define visualmente onde cada área fica na grade.
Os nomes ("header", "menu", "content", "footer") são arbitrários, mas devem ser consistentes com os usados em grid-area.

column-gap, row-gap e gap

column-gap: Define o espaço horizontal entre as colunas do grid.

column-gap: 20px;

row-gap: Define o espaço vertical entre as linhas do grid.

row-gap: 30px;

gap: É um atalho que junta os dois:

gap: 30px 20px; /* row-gap column-gap */
justify-items, align-items e place-items

justify-items: Alinha os itens horizontalmente (no eixo inline → esquerda/direita) dentro de cada célula do grid.

display: grid;
justify-items: center;
  • start: alinha à esquerda.
  • center: centraliza.
  • end: alinha à direita.
  • stretch (padrão): o item se estica para preencher a célula.

align-items: Alinha os itens verticalmente (no eixo block → topo/baixo) dentro das células.

display: grid;
align-items: end;
  • start: alinha ao topo
  • center: centraliza verticalmente
  • end: alinha à parte de baixo
  • stretch (padrão): o item se estica verticalmente pra preencher a célula

place-items: É um atalho (shorthand) para usar align-items e justify-items ao mesmo tempo.

display: grid;
place-items: center;

Exemplo prático completo:

display: grid;
grid-template-columns: repeat(2, 150px);
grid-template-rows: repeat(2, 100px);
place-items: center;
gap: 10px;

Esse código vai centralizar os itens tanto na horizontal quanto na vertical, dentro de cada célula do grid.

justify-content, align-content e place-content

Essas propriedades são parecidas com justify-items e align-items, mas têm um foco diferente: Elas alinham o grid inteiro dentro do container, e não os itens dentro das células.

justify-content: Alinha o grid como um todo no eixo horizontal (esquerda ↔ direita), dentro do container.

display: grid;
justify-content: center;

align-content: Alinha o grid no eixo vertical (cima ↕ baixo), dentro do container.

display: grid;
align-content: space-between;

Valores comuns:

  • start: alinha o grid à esquerda
  • center: centraliza o grid
  • end: alinha o grid à direita
  • space-between: distribui o espaço entre as colunas
  • space-around: espaço igual nas laterais internas
  • space-evenly: espaço totalmente uniforme em volta

place-content: É um shorthand que une align-content e justify-content.

display: grid;
place-content: center; /* centro vertical e horizontal */
place-content: start end; /* align-content: start; justify-content: end */
grid-auto-rows, grid-auto-columns e grid-auto-flow

Elas controlam o que acontece com os itens que não têm posição explícita definida — ou seja, quando você não especifica linhas ou colunas no grid-template-*, essas propriedades entram em ação.

grid-auto-rows: Define a altura padrão das linhas geradas automaticamente, quando você adiciona mais itens do que linhas previstas no grid-template-rows.

display: grid;
grid-auto-rows: 150px;

Exemplo: Se você tem grid-template-rows: 100px; e insere mais de 1 linha de itens, a primeira terá 100px e as próximas vão usar o valor de grid-auto-rows (150px).

grid-auto-columns: Define a largura padrão das colunas geradas automaticamente, caso o layout precise criar colunas extras.

display: grid;
grid-auto-columns: 200px;

Exemplo: você colocou os itens com grid-auto-flow: column, e o layout precisa criar novas colunas além das definidas — aí ele usa esse valor.

grid-auto-flow: Define a direção do preenchimento automático dos itens no grid:

display: grid;
grid-auto-flow: row; /* padrão */

Valores Comuns:

  • row (padrão): os itens são inseridos linha por linha
  • column: os itens são inseridos coluna por coluna
  • row dense ou column dense: o navegador tenta preencher os “buracos” do grid, se houver espaços sobrando

Position: Static

É o comportamento natural dos elementos em uma página web. Quando um elemento está com position: static (ou sem nenhuma posição definida), ele é exibido de acordo com o fluxo normal do HTML, ou seja, um abaixo do outro, da esquerda para a direita (em textos ou blocos).

Características principais:

  • É o valor padrão, ou seja, se você não definir nenhuma posição, o navegador assume static.
  • O elemento não pode ser movido usando top, right, bottom ou left. Esses valores não têm efeito com static.

Position: Relative

Quando um elemento tem position: relative, ele permanece no fluxo normal da página (ou seja, ocupa seu espaço original), mas pode ser movido usando as propriedades top, right, bottom e left.

Características principais:

  • O elemento mantém seu espaço original na página.
  • Pode ser deslocado em relação à sua posição original.
  • É muito útil quando você quer fazer pequenos ajustes na posição ou criar referência para elementos absolute.

Position: Absolute

Quando um elemento tem position: absolute, ele sai do fluxo normal da página, ou seja, não ocupa espaço como os elementos static ou relative.
Ele é posicionado relativo ao primeiro elemento ancestral (pai, avô, etc.) que tem position: relative, absolute ou fixed.
Se não encontrar nenhum ancestral com position definido, ele será posicionado relativo ao "html" (ou ao body).

Características principais:

  • Sai do fluxo: não afeta nem é afetado por outros elementos ao redor.
  • Pode ser posicionado com top, right, bottom e left.
  • Se move com base no elemento pai com posição definida (relative, absolute, fixed ou sticky).
Dica:

Use absolute quando você quiser colocar algo em cima, sobreposto, ou encaixado em um ponto específico de um container. Por exemplo:

  • Ícones no canto de uma imagem.
  • Balões de fala.
  • Pop-ups simples.

Position: Fixed

Um elemento com position: fixed é removido do fluxo normal da página e posicionado em relação à janela de visualização (viewport), ou seja, a tela do navegador. Ele não se move quando você rola a página, mantendo-se fixo no mesmo lugar.

Características principais:

  • Posicionado com base na viewport (não depende de elementos pai).
  • Usa top, right, bottom, left para se posicionar.
  • Permanece fixo mesmo ao rolar a página.
  • Não ocupa espaço no layout (sai do fluxo).

Position: Sticky

O sticky é uma mistura de relative com fixed:

  • O elemento começa como relative (segue o fluxo da página).
  • Mas, ao rolar a página até atingir um certo ponto (como top: 0), ele "gruda" na posição, como se fosse fixed.

Características principais:

  • Depende de um valor como top, left, right ou bottom pra funcionar.
  • O elemento só “gruda” dentro do limite do seu elemento pai.
  • Funciona bem para menus de seção, cabeçalhos de tabela, etc.
Dica:
  • O sticky precisa que o elemento pai tenha altura suficiente pra ele “rolar”.
  • Ele pode não funcionar se o pai estiver com overflow: hidden ou overflow: auto.
  • Ideal para títulos de seções, menus flutuantes dentro de áreas específicas e tabelas fixas.

Z-Index

O z-index define a ordem de empilhamento (stacking order) dos elementos no eixo Z (profundidade), ou seja, qual elemento fica na frente ou atrás de outro quando eles se sobrepõem.

Características principais:

  • Só funciona em elementos que têm position diferente de static (relative, absolute, fixed, ou sticky).
  • Quanto maior o valor do z-index, mais na frente o elemento fica.
  • Pode ter valores positivos, negativos ou zero.
Dica:
  • Se dois elementos têm o mesmo z-index, o que vier por último no HTML fica na frente.
  • Um z-index: -1 pode fazer um elemento ir atrás de tudo (até mesmo do fundo da página, dependendo do caso).
  • Usar z-index com sabedoria evita bugs visuais, como botões que não clicam porque outro elemento está invisivelmente "em cima".
